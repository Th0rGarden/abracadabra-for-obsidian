Master and command your Obsidian plugins.

Why should managing plugins feel like deciphering ancient scrolls? With Abracadabra, the secrets are revealed: every command, every note, every hidden feature—summoned into one unified, dynamic control deck.

No more context switching. No more feature blindness. Just pure, focused power—finally in your hands.

What is Abracadabra?

Abracadabra is a management console that fundamentally redefines your relationship with the Obsidian plugin ecosystem, elevating your ability to manage, comprehend, and command your tools. It also provides universal quality-of-life upgrades, like adding a search bar to other plugins' settings pages.

🔒 The Problem

The potential of many plugins is latent, locked away behind fragmented interfaces. Documentation lives on GitHub, commands in the palette, hotkeys buried in settings. This forces constant context switching, breaks your flow state, and leads to "feature blindness"—where you only leverage a fraction of a plugin's true power.

🪄 Enter Abracadabra

Abracadabra consolidates this disparate information into a single, coherent interface. It summons a dynamic Command Deck for each plugin—a powerful, unified workspace. This drastically reduces cognitive friction and helps you unlock the full potential of your tools.

The Three Views

Access Abracadabra's power anywhere in your workspace through three distinct views:

  1. Inline View: Integrates directly into any plugin's settings page.
  2. Pop-out Window: A floating, draggable, and resizable window for parallel work.
  3. Dedicated View: A full-pane view you can place anywhere in your workspace.

Installation

From Community Plugins (Coming Soon)

Abracadabra is awaiting approval to be listed in the official community plugins browser. Until then, please use the BRAT method below.

Using BRAT (Recommended)

BRAT (Beta Reviewer’s Auto-update Tool) is currently the best way to install the plugin and receive updates.

  1. Install the BRAT plugin from the Community Plugins browser in Obsidian.
  2. Enable BRAT in your settings under the "Community plugins" tab.
  3. Open the command palette (Ctrl/Cmd + P) and run the command BRAT: Add a beta plugin for testing.
  4. Paste the following repository path into the prompt: 
    Th0rGarden/Abracadabra
  5. Click "Add Plugin". BRAT will install it.
  6. Finally, go back to the Community plugins tab in your settings and enable "Abracadabra".

Core Concepts

Understanding a few key concepts will help you get the most out of Abracadabra.

The Command Deck

The central idea of Abracadabra: a unified interface created for *every other plugin* you have installed. Instead of hunting for information in different places, the Command Deck summons everything into one place: documentation, commands, notes, and more.

Components

The Command Deck is modular, built from a suite of powerful tools called "Components". Each serves a specific purpose. Show, hide, and reorder these components to create the perfect workspace for your needs.

Available components:

  • Info Card: The control panel for a plugin.
  • Readme Viewer: In-app documentation.
  • Command Module: A customizable grid of command buttons.
  • Commands & Hotkeys: A complete list of a plugin's commands.
  • Plugin Notes: A personal scratchpad for each plugin.
  • Console Logger: A live debugging tool.

Tip: Components are designed to work together. For example, use the Commands & Hotkeys list to discover a plugin's actions, then add your favorites to a custom Command Module for easy access. It's all about building the perfect interface for *you*.

The Layout System

A powerful two-tiered layout system gives you complete control over your workspace.

  • 🌍 Global Layout: In Abracadabra's main settings, define the master layout. This default order and visibility applies to all plugins.
  • 🔬 Per-Plugin Layout: When viewing a specific plugin, use the Abracadabra ˅ menu to override the global layout. Hide a component just for that plugin, or reorder them into a unique configuration, tailoring the Command Deck to the specific needs of each tool.

Customization Guide

Abracadabra is designed to be highly customizable, allowing you to tailor the interface to your exact preferences, both globally and on a per-plugin basis.

🌍 The Global Layout (Master Configuration)

Your primary customization hub is Abracadabra's own settings page (Settings → Abracadabra). Here you can:

  • Set the Global Component Order: Use the arrows in the Component Layout & Order card to define the default order in which components appear for all plugins. You can even move the plugin's native settings block.
  • Enable/Disable Components Globally: Toggle components on or off by default. For example, if you don't use the Console Logger, you can disable it here to hide it from all plugins.
  • Configure Component Defaults: Each component has its own set of default settings, such as whether it should be expanded or collapsed, default sizes, colors, and more.

🔬 The Per-Plugin Layout (The Controls Menu)

When you are viewing a specific plugin's Command Deck (in any of the three views), you have access to the Abracadabra ˅ menu in the Info Card header. This menu allows you to make overrides for that specific plugin.

  • Control Visibility: Temporarily show or hide components for the current plugin without changing your global defaults.
  • Reorder Layout: Change the order of components just for this plugin using a drag-and-drop interface right inside the menu.
  • Pin Section States: If you prefer manual control, you can disable "Auto-save section states" in the global settings. Then, you can use this menu to "pin" a section as expanded or collapsed, and it will always open that way for that specific plugin.
  • Reset Plugin Layout: This powerful option removes all per-plugin overrides (visibility, order, and pinned states), reverting it back to your global default settings.

Tip: Don't be afraid to experiment with per-plugin layouts! The Reset Plugin Layout button in the Controls Menu is your safety net. If you ever make a mess of a plugin's layout, one click will restore it to your global defaults.

Bonus Features & Universal Enhancements

Abracadabra doesn't just enhance individual plugin pages; it upgrades your entire Obsidian experience with powerful, universal features that work everywhere.

Search Any Plugin's Settings

Tired of scrolling through dozens of settings to find that one toggle? Abracadabra injects a powerful search bar directly into the native settings pages of other plugins. This is a universal upgrade to the entire Obsidian settings experience, eliminating tedious hunting and saving you time.

Quick Access with the Plugin Picker Status Bar

Enable the status bar item in settings to get a persistent ✨ icon in your Obsidian status bar. Clicking it opens a fuzzy-searchable list of all your enabled plugins, letting you instantly launch the Abracadabra pop-out window for any of them. The list intelligently sorts by frequency of use, and you can "star" your favorite plugins to pin them to the top.

Tip: Star your most frequently used or complex plugins in the Plugin Picker list. This pins them to the top, giving you one-click access to their Command Deck from anywhere in Obsidian.

Uncap Your Theme Picker

If you're a theme enthusiast, you've likely noticed that Obsidian's theme selection list gets cut off if you have too many installed. Abracadabra intelligently patches this modal, removing the item limit and allowing you to see and search your entire collection. It's a simple but profound fix for a common frustration, which can be toggled in Extras & Dev Tools.

Open any plugin settings with new commands

Abracadabra adds a new, powerful commands to the Obsidian command palette,"Open settings for another plugin...", which presents a fuzzy-searchable list of all your installed plugins. This allows you to jump instantly to any settings page from anywhere in your vault. Beats opening the settings manually, singing the ABCs as you look around for the plugin and finally click again to open them.

Power Combo: Add these commands to a Command Module to open settings for any plugin in a single click.

Enhance the Installed plugins list

The main "Community Plugins" tab in Obsidian's settings gets new powers. You can enable options in Extras & Dev Tools to add:

  • An Abracadabra Shortcut Button (✨) next to every installed plugin, allowing you to launch its pop-out window directly from the list.
  • A "Submit Issue" Button next to each plugin, providing another convenient shortcut for bug reporting.

Create a True Multi-Window Workspace

The pop-out window is more than just a pop-out. It's a fully independent, draggable and resizable window. Drag it by its header to position it anywhere on your screen, and pull its corners or edges to resize it to the perfect dimensions. This allows you to create custom, multi-monitor-like layouts, viewing documentation alongside settings or keeping a command grid handy while you work. Default size and position can be set in the Abracadabra Window Settings.

Discover with Intelligent Action Buttons

The action buttons in the Info Card are smarter than they look. Buttons like "Docs" and "Wiki" aren't just static links; they only appear if Abracadabra intelligently detects that the resource actually exists by scanning the plugin's files and its GitHub repository. This results in a cleaner, more relevant UI that adapts to each plugin's available resources.

Component: Info Card

The brain of the Command Deck. A mission control panel providing at-a-glance metadata and a powerful suite of one-click actions to manage, update, and learn about the plugin.

The Header

The top bar of the Info Card. Always visible, providing primary controls for the component and for Abracadabra itself.

  • Expand/Collapse: The entire left side of the header, containing the plugin's name, is a toggle. Clicking it expands or collapses the main content area below. A ▶ icon indicates the current state.
  • Abracadabra Controls (Right Side):
    • Abracadabra ˅ Menu: The master control menu for the current plugin's layout. Opens a dropdown to show/hide components, reorder them with a drag-and-drop interface, pin their state, or reset the layout to the global default.
    • Open in popout window Button (❐): (Standard settings view only) Instantly opens the entire Abracadabra interface for the current plugin in a separate, draggable, and resizable window.

Main Content Area

The collapsible body of the Info Card, containing detailed information and the main action buttons.

Identity & Status

Displays the plugin's core identity with intelligent enhancements.

  • Author & Description: The plugin's author and its official description. Abracadabra intelligently attempts to find and link the author's name to their GitHub profile, fostering a stronger connection.
  • Status Indicators: Subtle labels in the header identify the plugin's source: (BRAT) for BRAT installs or (Sideloaded) for custom/manual installs, giving you immediate context. A prominent yellow banner appears if the plugin has been removed from the community store.
Version Management

A dynamic line giving a complete, real-time overview of the plugin's version status.

  • Installed Version: Your current version number.
  • Status Text: A message indicating the current status: ✓ Up to date, Update to [version] available, Beta [version] available, or an error state.
  • Update Buttons: Appear when an update is found for one-click installation of stable or beta releases.
  • Release Notes (📜): Opens a modal window with changelogs for recent versions.
  • Refresh (🔄): Forces an immediate re-check for new versions, bypassing the cache.
The Action Bar

Your primary toolkit for interacting with the plugin and its developer, with buttons grouped by function.

  • Management & Support:
    • 🔌 Enable/Disable Plugin: Toggles the plugin on or off.
    • 💖 Donate: A smart button that scans the manifest and README for all funding links. Becomes a dropdown menu if multiple platforms (Ko-fi, Patreon, etc.) are found.
  • Intelligent Exploration & Learning:
    • 🐙 GitHub: Direct link to the source code.
    • 📖 Docs: This button is smart. It actively scans the plugin's README for keywords like "documentation," "docs," or "guide" and only appears if a relevant link is found, reducing UI clutter.
    • 📚 Wiki: This button uses an advanced detection system. Instead of just checking for a standard URL, it scans the plugin's main GitHub page to see if a Wiki tab is actually present, ensuring the button only appears when a wiki genuinely exists.
    • 📺 YouTube: This is a dynamic search query generator that uses the current plugin's name to find relevant tutorials and showcases on YouTube.
  • Troubleshooting & Development:
    • 🐛 Submit Issue: Opens a new GitHub issue, pre-filled with essential debug info. See the FAQ for the workflow.
    • 📋 Copy Console Logs: One-click copy of logs from the Console Logger for this specific plugin.
    • 🔧 Dev (Menu): A dropdown with power-user tools like Reload Plugin, Open Plugin Folder, Add .hotreload file, and Reveal/View `data.json`. This menu is fully configurable in settings.

Power Combo: The Abracadabra ˅ menu in the header is the central nervous system for the entire Command Deck. It allows you to customize the visibility and order of all other components on a per-plugin basis, transforming the interface to perfectly match your needs for each specific tool.

Component: Controls Menu

The Abracadabra ˅ menu, located in the header of the Info Card, is the master control panel for customizing the layout and behavior of the Command Deck on a per-plugin basis. It allows you to override your global settings for specific plugins, tailoring the interface to your exact needs for each tool.

Navigation

  • Open Abracadabra Settings: A quick shortcut to jump directly to Abracadabra's main settings page.
  • Open in popout window: Launches the entire Command Deck for the current plugin into a separate, draggable, and resizable window.
  • Open in new tab: Opens the Command Deck in a dedicated, full-pane Dedicated View within your Obsidian workspace.

Visible Components

This section of the menu contains a list of all available components (Readme, Command Module, etc.) with a checkbox next to each one. Use these checkboxes to show or hide components for the current plugin, overriding your global visibility settings.

  • Unchecking a component hides it only for the plugin you are currently viewing.
  • This allows you to create minimalist layouts for simple plugins while keeping all tools visible for more complex ones.

Component Order

A dynamic, drag-and-drop list that allows you to reorder the components for the current plugin. Simply click and drag any component (including the "Plugin's Native Settings" block) to a new position to create a custom layout.

Tip: This is incredibly powerful for prioritizing information. For a plugin you use for its commands, drag the Command Module to the top. For a plugin you're still learning, move the Readme up.

Pin Section States

This group of options only appears if you have disabled "Auto-save section states" in the global settings. It gives you manual control over which sections are expanded or collapsed by default for the current plugin.

  • Clicking a component in this list will "pin" its current state (e.g., if the Readme is currently expanded, clicking "Readme" will pin it as expanded).
  • A pinned icon (📌) will appear next to the component, indicating its state is saved. Click it again to unpin.

Reset

  • Reset Plugin Layout: This is your escape hatch. Clicking this option will remove all per-plugin overrides you've made—visibility, order, and pinned states—and instantly revert the layout for the current plugin back to the global defaults you configured in Abracadabra's main settings.

Component: Readme Viewer

Brings a plugin's full documentation directly into your workspace, so you can learn how to use a tool while you're configuring it.

Header & Controls

The header provides the title and a suite of controls to customize your reading experience.

  • Search Bar: Full-text search to find any keyword. Highlights all matches and provides `↑`/`↓` buttons and a match count (`1/15`) for easy navigation.
  • Font Controls: `A+`, `A-`, and `Reset` buttons to adjust font size.
  • Refresh (🔄): Force-fetches the latest README from its source, bypassing the cache.
  • Expand/Collapse: Click the title to show or hide the Readme content.

Content & Rendering

  • No More Broken Media: Automatically fixes relative image paths from GitHub and embeds YouTube video links, ensuring all media renders correctly.
  • Configurable Appearance: Global settings allow a "borderless" style for a cleaner look and a choice between full content height or a capped, scrollable box.

Intelligent Fetching & Workflows

The viewer is smart about where it gets documentation, enabling powerful workflows:

  1. First, it tries to fetch the latest version from GitHub.
  2. If offline, it falls back to a local cache.
  3. As a final fallback, it loads the README.md file from the plugin's folder in your vault.

Power Combo: Live Documentation Preview
The local file fallback is a game-changer for developers. If you are editing a plugin on your machine, Abracadabra automatically detects and displays your local README.md file. This allows you to write and preview your documentation in real-time inside Obsidian without pushing to GitHub.

Component: Command Module

Turns any plugin into a custom-built app. Build a visual grid of buttons for your most-used actions, creating a personal, intuitive interface for any tool.

Build Your Own Interface

Design the perfect launcher.

Click an empty tile to add a new command, not just from the currently active plugin but any plugin in Obsidian. 🎨 Fully customize the tile's label, choose from over 1000 Lucide icons, and set unique background/text colors.

Build cross-plugin dashboards (e.g., a "Writing" board with commands from three different plugins).

Enter Edit mode and access:

  • Board Settings (⚙️): Configure board-specific defaults like grid dimensions (rows/columns) and default tile/text colors in the Command Module Settings.
  • Clear Board (🗑️): Wipes the grid clean with one click.
  • Pre-populate (🪄): A powerful bridge to the Commands & Hotkeys component. Instantly fill the board with the plugin's commands. If the "Show commands from other plugins" checkbox is checked in the Commands & Hotkeys component, those related commands will be included too.
  • Source Plugin Indicator: If a tile's command comes from a different plugin, a small label appears at the bottom indicating its source.

    • Power User Tip: Create a "master" Command Module for a simple utility plugin you rarely configure. Fill it with your most-used commands from other plugins (like Templater, QuickAdd, etc.) to create a universal, cross-plugin command palette you can pop out anytime.

    Board Management & Integration

    Header controls give you full power over the board's structure and content.

    Power Combo: Building Integrated Dashboards
    The Pre-populate action becomes even more powerful when combined with the Commands & Hotkeys component. Check the "Show commands from other plugins" box in that component first, and then click Pre-populate. Abracadabra will fill your board not only with the plugin's own commands but also with related commands from other plugins, making it easy to build a truly integrated workflow dashboard.

    Component: Commands & Hotkeys

    Discover every hidden feature. Lists all of a plugin's commands, letting you test-drive them instantly and manage hotkeys without leaving the page.

    Discovery & Execution

    • Run Any Command (▶️): A "Run" button next to every command triggers it instantly. Perfect for learning a new plugin's capabilities without committing to a hotkey.
    • Hotkey Display: Shows the currently assigned hotkey for each command, or "Not set".
    • Configure Hotkeys Button: A master button in the header takes you directly to Obsidian's hotkey settings, pre-filtered for the current plugin.

    Smart Filtering & Integration

    • Complete Command Registry: Displays every command registered by the plugin, sorted alphabetically.
    • Uncover Hidden Integrations: A checkbox in the header expands the list to show commands from *other* plugins that mention the current plugin's name.
    • Source Plugin Indicator: For commands from other plugins, a small label indicates which plugin it belongs to.

    Power Combo: Discover and Build
    The "Show commands from other plugins" checkbox is a powerful discovery tool. When checked, it not only reveals integrations here but also tells the Command Module's Pre-populate action to include these external commands, helping you build cross-plugin dashboards with one click.

    Component: Plugin Notes

    A personal, persistent scratchpad for every plugin, fully integrated into your vault.

    Editor Features

    • Auto-Save: Notes save automatically after a configurable delay. A manual save button and status indicator (`Unsaved changes`, `Saving...`, `Saved!`) are also provided.
    • Resizable & Remembers: Drag the bottom edge to resize the notes area. This height can be remembered on a per-plugin basis.
    • Auto-Collapse: The section can automatically hide itself if the note file is empty, reducing visual clutter.

    Plugin Notes are standard Markdown (.md) files inside your vault, in the folder specified by the Notes storage path setting.

    This deep integration means your plugin notes benefit from Obsidian's core features:

    • Globally Searchable: Find notes using Obsidian's built-in search.
    • Linkable: Link to and from your plugin notes like any other file.
    • Graph View: Notes appear as nodes in the graph, connected to any linked notes.
    • Tags & Dataview: Use tags and query your notes with the Dataview plugin.

    Tip: You can link to specific headings within your Plugin Notes from anywhere in your vault. For example, create a link like [[MyPlugin plugin notes#My Custom Snippet]] to jump directly to a specific code block or setup instruction you've saved.

    Component: Console Logger

    A live, in-app debugging tool for power users and developers. Captures and displays console messages without needing the Obsidian Developer Tools.

    Live Debugging Feed

    • Live Capture: Intercepts and displays console.log, warn, error, info, and debug messages as they happen.
    • Source Filtering: Automatically identifies the source plugin for each log. A dropdown in the header lets you filter for the current plugin's logs or view all captured logs. This is essential for isolating issues.
    • Intuitive Interface: Each log entry is timestamped and color-coded by severity (e.g., errors in red, warnings in yellow), making the feed easy to parse at a glance.

    The Bug-Squashing Superpower

    This component is the engine behind Abracadabra's streamlined bug reporting. The header contains one-click buttons to Copy all currently displayed logs to the clipboard or Clear the feed.

    Power Combo: The Ultimate Bug Reporting Workflow
    When a developer asks for logs, your workflow is simple:

    1. Enable the Console Logger component (via the Abracadabra ˅ menu in the Info Card).
    2. Perform the action that causes the bug. The logger will capture the error.
    3. Go to the Info Card and click the `📋 Copy Console Logs` button.
    This copies the perfectly filtered logs for that specific plugin, ready to be pasted into a GitHub issue created with the `🐛 Submit Issue` button.

    Architectural Philosophy

    ► For those curious about the engineering...

    Abracadabra is built with performance and robust engineering in mind:

    • Dynamic Injection: A MutationObserver ensures the UI is injected seamlessly without flickering.
    • Asynchronous Loading: Heavy operations are deferred until onLayoutReady to ensure Obsidian's startup remains fast.
    • Component-Based Architecture: A builder pattern manages the rendering of UI components based on global and per-plugin settings.
    • Resilient Update Engine: A multi-source system (Community List, Plugin Update Tracker API, and direct GitHub API) with retry logic and exponential backoff ensures reliability.
    • Custom Settings UI: The settings panel is built with raw DOM manipulation for a card-based, theme-aware user experience.
    • Advanced Data Handling: Features multi-layered caching (in-memory Maps and data.json), intelligent repository detection for all plugin types, and README pre-processing for rich media.
    • Console Patching: Safely patches the global console object to capture logs without interfering with the Developer Console.

    Troubleshooting & Errors

    Q: The "Update" button failed or is not appearing. What do I do?

    This can happen for a few reasons:

    • Network Issues: The most common cause. Abracadabra needs an internet connection to check GitHub for updates. Check your connection and try clicking the refresh icon (🔄) in the Info Card.
    • GitHub Rate Limits: If you check many plugins quickly, you might hit GitHub's API rate limit. The error message will usually mention this. The best solution is to add a GitHub Personal Access Token in Abracadabra's "Info Card" settings. This significantly increases your limit.
    • API Issues: Abracadabra uses an external API for more reliable version checking. If this API is down, it may affect update checks. This is usually temporary.
    • No Repository Link: If a plugin's developer hasn't included a repository link in their manifest.json, Abracadabra cannot find where to check for updates. This is common for very new or unlisted plugins.
    • Cache Issues: In rare cases, the version cache might be stale. Go to Settings → Abracadabra → Data Management and click "Clear all release info cache," then return to the plugin's page and refresh.
    Q: The Readme isn't loading or looks strange.

    Here's how to troubleshoot a problematic Readme:

    • Offline: If you're offline, Abracadabra will try to load a cached version or a local file. If neither exists, it can't load.
    • Unusual File Name: Abracadabra looks for README.md or readme.md. If the developer used a different name, it won't be found.
    • Private Repository: The plugin cannot access documentation from private GitHub repositories.
    • Force Refresh: Click the refresh icon (🔄) in the Readme Viewer's header to force a fresh download from GitHub, which often resolves temporary network glitches.
    Q: Why isn't the "Docs" or "Wiki" button showing up for a plugin?

    These buttons are smart and only appear when they have something to link to.

    • The Docs button appears if Abracadabra finds a link with keywords like "documentation," "docs," or "guide" in the plugin's README file.
    • The Wiki button appears only if the developer has enabled the "Wiki" feature on their GitHub repository.
    If a button is missing, it means the resource likely doesn't exist or isn't linked in a way the plugin can detect.

    Functionality & Behavior

    Q: How do I reset the layout for just one plugin?

    Easy! Go to the settings page for the plugin you want to reset. In the Info Card header, click the Abracadabra ˅ control menu and select "Reset Plugin Layout". This will remove all per-plugin overrides (visibility, order, and pinned states) and revert it to your global default settings.

    Q: Why is a plugin marked as "(Sideloaded)" or "(BRAT)"?

    These indicators give you quick context about how a plugin was installed.

    • (BRAT): The plugin was installed using the BRAT (Beta Reviewer's Auto-update Tool). Abracadabra will use GitHub directly for updates.
    • (Sideloaded): The plugin is not found in the official community plugin list and is not recognized as a BRAT install. This is normal for plugins you are developing locally or have installed by manually placing them in your vault's .obsidian/plugins folder.
    Q: Can I search for commands outside of a plugin's settings page?

    While the visual list of commands is part of the Command Deck, you can always use Obsidian's built-in Command Palette (Ctrl/Cmd + P) to search for and run any command from any plugin at any time. Abracadabra's "Commands & Hotkeys" component is designed for discovery, learning, and testing commands in context.

    Q: Where are my "Plugin Notes" stored? Can I access them elsewhere?

    Yes! Your notes are just standard Markdown files stored inside your vault. You can find the exact folder path in Settings → Abracadabra → Plugin Notes Settings → Notes storage path. Because they are regular notes, you can link to them, search for them using Obsidian's global search, and even query them with Dataview, just like any other file in your vault.

    Bugs & Support

    Q: How do I report a bug for a different plugin using Abracadabra?

    Abracadabra makes this easy and helps developers immensely.

    1. Go to the settings page of the plugin with the bug.
    2. In the Info Card, click the "Submit Issue" (🐛) button. This opens a new GitHub issue and pre-fills it with essential system information.
    3. If the developer asks for logs, enable the Console Logger component, reproduce the bug, and then click the "Copy Console Logs" (📋) button in the Info Card. Paste this into your GitHub issue.

    Q: How do I report a bug for Abracadabra itself?

    We appreciate it! The best way is to use Abracadabra on itself. Go to Settings → Abracadabra and use the “Submit Issue” button in its own Info Card. This gives us the same high-quality debug information we help you provide to other developers!
